Default: 0px
Accepts: width in px / any fixed units
This is the border's size

--aug-border-bg
Default: none
Accepts: same as css background

--aug-border-opacity
Default: 1
Accepts: number between 0 and 1 inclusive with no unit

--aug-inset
Default: 0px
Accepts: distance in px / any fixed units
This is the distance between the inner edge of the border and the inset bg itself

--aug-inset-bg
Default: none
Accepts: same as css background

--aug-inset-opacity
Default: 1

/// fallback values for browsers without clip-path or -webkit-clip-path (pre-chromium Edge) ///
--aug-border-radius-fallback
Default: an automatic calculation based on size of any corresponding corner aug equipped to the element.
Accepts: same as css border-radius
Note: augs along the edges are not rendered or considered in pre-chromium Edge at all, only corner augs are simulated

--aug-border-radius-fallback-inset
Default: an automatic calculation based on size of any corresponding corner aug equipped to the element, minus the (border width + inset distance).
Accepts: same as css border-radius

--aug-border-fallback-color
Default: the value of --aug-border-bg, else transparent
Accepts: color value
Note: if --aug-border-bg is not suitable as a border-color value, the border will not draw unless this fallback is set
The border is drawn on a pseudo element like the full-support one is, so it will not upset layout if it is drawn or not
